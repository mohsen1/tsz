# TS2323 vs TS2528: Duplicate Default Export Errors

## Issue

We emit **TS2528** ("A module cannot have multiple default exports") for all duplicate default export cases, but TSC is more granular and emits **TS2323** ("Cannot redeclare exported variable") for specific patterns.

## Example

```typescript
// exportDefaultInterfaceAndTwoFunctions.ts
export default interface A { a: string; }
export default function() { return 1; }
export default function() { return 2; }
```

**TSC emits:**
- TS2323 (cannot redeclare exported variable)
- TS2393 (duplicate function implementation)

**We emit:**
- TS2528 (module cannot have multiple default exports)

## Root Cause

Location: `crates/tsz-checker/src/import_checker.rs:519-527`

```rust
// TS2528: Check for multiple default exports
if export_default_indices.len() > 1 {
    for &export_idx in &export_default_indices {
        self.error_at_node(
            export_idx,
            diagnostic_messages::A_MODULE_CANNOT_HAVE_MULTIPLE_DEFAULT_EXPORTS,
            diagnostic_codes::A_MODULE_CANNOT_HAVE_MULTIPLE_DEFAULT_EXPORTS,
        );
    }
}
```

We detect the issue correctly but emit a general error code instead of being specific about what's being redeclared.

## Required Fix

Need to:
1. Add TS2323 diagnostic code
2. Distinguish between different types of default exports (value, class, function, interface)
3. Emit TS2323 for redeclared exported variables/values
4. Emit TS2393 for duplicate function implementations
5. Keep TS2528 for other cases

This is more complex than initially estimated because it requires:
- Analyzing the export declaration type
- Tracking which type of default export already exists
- Choosing the appropriate error based on the combination

## Impact

- Affects 9 tests
- Category: "Wrong error code" (not "not implemented")
- Complexity: Medium-High
- Value: Low-Medium (we already detect the error, just wrong code)

## Recommendation

**Defer** - This is a wrong error code issue, not a missing feature. The error is detected and reported, just with a different code. Focus on:
1. Truly NOT IMPLEMENTED error codes (like TS7005)
2. Missing error checks (all-missing category)
3. High-impact false positives with clear fixes
