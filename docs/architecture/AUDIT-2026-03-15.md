# Architecture Audit — 2026-03-15

**Scope**: Last 100 commits on `main`, full codebase scan against NORTH_STAR.md principles
**Codebase size**: 946K lines of Rust across 7 crates
**Conformance**: 10,799 / 12,572 (85.9%)

---

## Executive Summary

The codebase is in strong architectural shape. The NORTH_STAR corrections (A–F) are all enforced, `TypeKey` is sealed behind `TypeData`, architecture contract tests guard boundaries, `query_boundaries` mediate checker→solver access, and narrowing properly delegates to solver. The binder is completely clean — zero solver/checker imports, zero type computation.

The findings below are refinement opportunities, not fundamental violations.

---

## Findings

### F1. Private `type_queries::data` Usage in Checker (P1 — High)

The checker accesses the solver's private `type_queries::data` submodule directly in 16+ call sites across 6 files, bypassing `query_boundaries`.

| File | Private API Used | Calls |
|------|-----------------|-------|
| `types/computation/access.rs` | `data::get_union_members()`, `data::get_intersection_members()` | 8 |
| `checkers/iterable_checker.rs` | `data::get_call_signatures`, `data::get_function_shape` | 2 |
| `checkers/jsx_checker.rs` | `data::contains_type_parameters_db()` | 1 |
| `assignability/assignability_checker.rs` | `data::get_enum_member_type()` | 2 |
| `assignability/assignment_checker.rs` | `data::get_union_members()` | 1 |
| `error_reporter/core.rs` | `data::get_intersection_members()` | 2 |

**Why it matters**: These create tight coupling to solver internals. When the solver's internal data representation changes, all these call sites break — exactly what the boundary layer prevents.

**Fix**: Create or extend `query_boundaries` helpers that wrap these private APIs. Add architecture contract test to flag `type_queries::data::` imports in checker code.

### F2. Declaration Emitter Helpers — God Object (P2 — High)

`crates/tsz-emitter/src/declaration_emitter/helpers.rs` is **7,618 lines** with **235 functions** — the single largest non-test, non-data source file. It mixes:

- Expression/node emission
- JSDoc parsing and emission (20+ functions)
- JS CommonJS export analysis (10+ functions)
- Namespace object analysis (5+ functions)
- Static method augmentation detection
- Import planning utilities

**Fix**: Split into focused modules:
- `jsdoc_emission.rs` — JSDoc parsing, template params, callback aliases
- `commonjs_analysis.rs` — CJS export collection, expando declarations
- `namespace_analysis.rs` — namespace object statements, reexports
- `helpers.rs` — remaining core emission utilities

### F3. Residual TypeData Matching in Checker (P3 — Medium)

Two places in checker production code directly match on `TypeData`, violating the sealed type principle:

**1. `error_reporter/operator_errors.rs:546–550`**
```rust
if matches!(
    self.ctx.types.lookup(type_id),
    Some(tsz_solver::TypeData::Enum(_, _))
) {
    return type_id;
}
```
**Fix**: Use `tsz_solver::visitor::is_enum_type()` (already exists).

**2. `types/computation/complex.rs:194–199`**
```rust
match db.lookup(ty) {
    Some(tsz_solver::TypeData::ReadonlyType(inner) | tsz_solver::TypeData::NoInfer(inner)) => {
        should_preserve_contextual_application_shape(db, inner)
    }
    _ => false,
}
```
**Fix**: Use `tsz_solver::visitor::readonly_inner_type()` and `tsz_solver::visitor::no_infer_inner_type()` (both exist).

Three additional `.lookup()` calls in checker code are inside `tracing::trace!()` macros for debug output — these are acceptable.

### F6. Duplicated Type Predicates Across Layers (P6 — Low)

`is_string_like_type` / `is_string_like` exists in four places:

| Location | Function |
|----------|----------|
| `checker/error_reporter/operator_errors.rs:208` | `is_string_like_type()` |
| `solver/relations/compat_overrides.rs:407` | `is_string_like()` |
| `solver/operations/binary_ops.rs:851` | `is_string_like()` |
| `solver/operations/call_args.rs:21` | `is_string_like_type()` |

Per NORTH_STAR §8.5, these should have a single source of truth in solver with checker importing it.

**Fix**: Consolidate into one `pub fn` in `solver/type_queries` and remove the rest.

## Architecture Contract Test Gaps

The existing tests are strong but could be extended:

1. **Flag `.lookup()` in checker code** outside `query_boundaries` and `tracing` macros
2. **Flag `type_queries::data::` imports** in checker code
3. **Flag `RelationCacheKey` construction** outside centralized helpers
4. **Flag `is_string_like` duplication** across modules
5. **Enforce file size limits** (fail CI if checker file exceeds 2,500 lines)
