## Intent

Fix the picked fingerprint-only conformance failure
`TypeScript/tests/cases/conformance/controlFlow/exhaustiveSwitchStatements1.ts`.
tsc and tsz already agree on diagnostic codes (`TS2339`, `TS2367`, `TS7027`);
the remaining gap is one or more diagnostic fingerprints. This slice will
identify whether the mismatch is a type display, diagnostic anchor, or ordering
issue and fix the owning checker/reporter path rather than suppressing the
fixture.

The root cause was that checker statement traversal reused the same unreachable
diagnostic path for branches skipped by statically true/false `if` conditions.
tsc still type-checks those branches, but it does not emit TS7027 solely because
the condition makes the branch unreachable. This caused extra TS7027 fingerprints
for the `else` branch in `if (!!true)` and the then branch in `if (!true)`.
